## WeChat Group Channel Fixes

This release focuses on the personal WeChat group channel and fixes several issues found during real group testing:

- **Accurate connection state**: The channel now distinguishes sidecar startup, QR-code readiness, successful login, and real connection. Channel management no longer shows "connected" before the account is actually logged in.
- **QR-code status reporting**: The Web API now exposes WeChat group QR status, QR image data, room list, login state, and startup error messages so the channel management UI can show the real scan-login flow.
- **Non-fatal sidecar errors after login**: Sidecar errors that arrive after login or after rooms are available are logged as non-fatal warnings instead of resetting the channel to a failed state.
- **Manual group selection scope**: `wechat_group_room_ids` now has priority over group names. Messages from unselected rooms are skipped before group-name fallback, while selected room IDs can enter group context without depending on the old `group_name_white_list`.

## Image Understanding in WeChat Groups

- **Direct image analysis**: When a user @mentions the bot with an image, the channel can inject a `<wechat-group-image>` context block generated by the vision tool.
- **Recent-image follow-up support**: Text requests such as "识别这张图" and "我发的图是啥内容" can resolve to the latest relevant group image.
- **Result caching**: Image understanding summaries are cached for the configured time window to reduce duplicate vision calls.
- **Controlled free-reply image understanding**: Non-@ image messages remain archived by default; they only enter the free-reply image-understanding path when the dedicated switch is enabled.

## Image Generation Routing

- **More natural Chinese image prompts**: The default image-generation prefixes now include common forms such as `生成一张`, `生成一个`, `生成一幅`, `生图`, `出图`, `绘制`, `画张`, and `画个`.
- **Common typo tolerance**: Prompts starting with `花张`, `花个`, `花一张`, `花一个`, or `花一幅` are treated as image-generation requests, avoiding fallback to normal Agent chat.
- **Pinned custom provider support**: Image generation now preserves the configured `provider=custom:<id>` and model when dispatching to the image-generation skill.
- **Invisible character cleanup**: Provider and model values are cleaned at the Web model-setting layer, channel layer, environment-sync layer, and image-generation script layer to avoid failures caused by pasted zero-width characters.
- **Python 3.9 compatibility**: The image-generation script now enables postponed annotation evaluation for compatibility with Python 3.9 deployments.

## Diagnostics

Image generation now logs the key path from queue to send:

- request queued
- request dispatched
- request handling started
- generation script started and exited
- failure details or first successful image path
- WeChat group image send attempt

These logs make it easier to tell whether a failure happened during trigger recognition, API invocation, script execution, or message sending.
