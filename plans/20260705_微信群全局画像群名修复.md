# 微信群全局画像群名修复

## 目标

修复 Web 控制台“全局画像”中“出现过的群”优先展示 `room_id` 的问题，改为尽量展示群名；同步优化微信群相关页面中已有群名但仍回退到群 ID 或占位文案的展示路径。

## 根因

- 全局画像的 `room_summaries` 来自 `wechat_group_profile_name_records`，历史记录可能只有 `room_id`、没有 `room_name`。
- Web 控制台部分房间列表只使用运行态 `rooms`，没有使用 `selected_room_names` 作为兜底。
- 当后端没有补齐群名时，前端详情标题直接使用 `room_id`。

## 实施步骤

- [x] 增加失败测试：画像记录缺少 `room_name` 时，服务应从归档消息补齐群名。
- [x] 增加失败测试：Web 画像接口应把已保存的 `wechat_group_names` 用作群名兜底。
- [x] 后端补齐群名解析：优先记录内群名，其次归档消息/回复，其次配置或运行态房间列表。
- [x] 前端优化显示兜底：已保存群名参与记忆/画像房间列表，画像详情无群名时不把群 ID 当标题。
- [x] 更新 `CHANGES.md`。
- [x] 运行相关单测与静态检查。
- [x] 提交并推送 Git。

## 完成记录

已完成以下改动：

- `channel/wechat_group/wechat_group_archive.py`：新增 `find_room_name()`，从消息和助手回复归档中读取最近的非空群名。
- `channel/wechat_group/wechat_group_profile_service.py`：全局画像 `room_summaries` 缺少 `room_name` 时，从归档补齐。
- `channel/web/web_channel.py`：`/api/wechat-group/memories/profiles` 返回前补齐缺失群名，配置中的 `wechat_group_names` 和运行态房间列表都作为兜底来源。
- `channel/web/static/js/console.js`：微信群记忆/画像房间列表使用已保存群名兜底；画像详情没有群名时显示“未命名群”，群 ID 保留为诊断信息。
- `tests/test_wechat_group_profile_service.py`、`tests/test_wechat_group_web.py`、`tests/test_wechat_group_memory_ui.py`：新增回归测试。
