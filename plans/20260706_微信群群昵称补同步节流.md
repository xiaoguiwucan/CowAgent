# 微信群群昵称补同步节流

## 目标

为微信群 `@` 成员时的“群昵称补同步”增加按群维度的分钟级冷却，默认每 `1` 分钟最多补同步一次；同时在 Web 控制台“群聊 -> 基础设置”中提供可配置项，并确保保存后当前运行态立即生效。

## 设计结论

- 节流粒度按 `room_id`，因为 `room.sync()` 刷新的是整群成员数据。
- 配置项命名为 `wechat_group_alias_sync_cooldown_minutes`，默认值 `1`。
- 配置既写入 `config.py` / `config-template.json`，也通过 Python `send_text` 命令实时传给 sidecar，避免用户保存后必须重启通道。
- sidecar 仅在 `room.alias(contact)` 为空时检查冷却；命中冷却则直接退回当前可用名称，不触发 `sync()`。
